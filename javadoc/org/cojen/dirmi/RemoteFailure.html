org.cojen.dirmi

Annotation Type RemoteFailure


  • @Documented
@Retention(value=RUNTIME)
@Target(value={METHOD,TYPE})
public @interface RemoteFailure
    Annotate a remote method or interface to specify what exception is to be used to indicate a remote call failure. By default, it is RemoteException, and it must be declared to be thrown. 
     @RemoteFailure(exception=SQLException.class)
 int executeUpdate(String sql) throws SQLException;
    Author:
    Brian S O'Neill

      boolean declared

    • Element Detail

      • exception

        public abstract Class<? extends Throwable> exception
        Specify the exception to throw when the remote call fails. If not RemoteException (or a superclass), then it must support exception chaining. A public constructor must exist which accepts a single argument of RemoteException or a superclass.
        Default:
        java.rmi.RemoteException.class

      • declared

        public abstract boolean declared
        By default, if the exception is checked, it must be declared to be thrown. Set to false to allow checked exception to be thrown without being declared.
        Default:
        true
